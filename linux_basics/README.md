# Linux Basics 🐧

This section covers the **fundamental concepts of Linux** required for DevOps engineers, system administrators, and cloud professionals.

Linux is widely used in **servers, cloud platforms, and containerized environments**, making it an essential skill for DevOps roles.

---

## 📌 What is Linux?

* Linux is an **open-source operating system kernel** created by Linus Torvalds in 1991.
* It powers most modern infrastructure including cloud servers, container platforms, and DevOps tooling.

### Key Features

* Open source
* Multi-user
* Multi-tasking
* Secure and stable
* Highly customizable

---

## 🏗 Linux Architecture

Linux architecture consists of several layers:

1. **Hardware** – Physical components (CPU, RAM, Disk)
2. **Kernel** – Core of the operating system
3. **System Libraries** – Functions used by programs
4. **Shell** – Interface between user and kernel
5. **Applications** – Programs used by the user

---

## 🐧 Linux Distributions

A Linux distribution (distro) is a packaged version of Linux that includes the kernel, system tools, and package manager.

Common Linux distributions used in DevOps:

* Ubuntu
* Debian
* CentOS
* Red Hat Enterprise Linux (RHEL)
* Fedora
* Amazon Linux

---

## 🚀 Why Linux is Important for DevOps

Linux is the backbone of modern infrastructure and DevOps environments.

Reasons:

* Most **cloud servers run on Linux**
* DevOps tools like **Docker, Kubernetes, Jenkins, and Terraform** run on Linux
* Enables powerful **automation through shell scripting**
* Provides strong **system control and troubleshooting capabilities**
